# Team Workspace Fix Report

## Problem

Team plan users could open Team Workspace pages but saw:

> "No workspace found. Contact support."

This happened because `requireTeamAccess()` returned a 404 when no workspace row existed, **before** any auto-creation logic ran.

---

## Root Cause

1. `requireTeamAccess()` checked for a workspace and immediately returned 404.
2. `ensureTeamWorkspaceForUser()` was only called **after** access was granted (too late).
3. Existing team users who signed up before workspace auto-create had no migration path on page load.

---

## Solution

### 1. Auto-create on every team API request

`requireTeamAccess()` now:

- Detects team/admin users with no workspace
- Calls `ensureTeamWorkspaceForUser()` automatically
- Repairs missing owner membership rows
- Returns `503 WORKSPACE_INITIALIZING` with a friendly message if creation is still in progress
- **Never** returns "No workspace found. Contact support."

### 2. Safer workspace creation

`ensureTeamWorkspaceForUser()` and `createWorkspaceForUser()` now:

- Check `workspaces WHERE owner_id = user.id` before inserting (duplicate safety)
- Handle unique constraint conflicts gracefully
- Add owner to `workspace_members` if missing (`ensureOwnerMembership`)
- Throw user-friendly errors instead of raw database messages

### 3. Existing team user migration

On any team page/API load, team users without a workspace get one created automatically with:

| Field | Value |
|---|---|
| `workspaces.name` | `{full_name}'s Workspace` |
| `workspaces.slug` | `{slugified-name}-{userId-prefix}` |
| `workspaces.owner_id` | user id |
| `workspace_members.role` | `owner` |

### 4. UI loading states

- Shows **"Creating your workspace..."** during initialization
- Auto-retries every 2 seconds until workspace is ready
- Never surfaces raw Postgres/Supabase errors

### 5. Team scripts

Team Scripts page now lists **all scripts generated by workspace members**, not only shared scripts. Shared scripts show a "Shared" badge.

## Flows Verified

| Scenario | Behavior |
|---|---|
| Existing team account, no workspace | Auto-created on first team page load |
| New team signup | Workspace created in `signupWithEmailPassword` |
| Free/Pro → Team upgrade | Workspace created in `updateUserPlan` (Stripe webhook) |
| Duplicate creation attempt | Returns existing workspace (no duplicate rows) |
| Owner without member row | Repaired via `ensureOwnerMembership` |
| Invited member (non-team plan) | Uses existing workspace membership |

## Database Requirements

Ensure these tables exist (from `supabase/schema.sql` + `supabase/team-workspace.sql`):

- `public.workspaces`
- `public.workspace_members`
- `public.workspace_invitations`
- `scripts.workspace_id`, `scripts.shared_with_workspace`

No new migration required for this fix.
